#### 💜 Bank Churn Predictor
```
STATUS: ACTIVE
ACCESS: PUBLIC
```
The Bank Churn Predictor is a machine learning project designed to predict whether a bank customer is likely to leave (churn) or continue using the bank’s services. The goal was to help banks identify at-risk customers early and improve customer retention strategies.

I started by working with a customer dataset containing features such as credit score, age, account balance, tenure, number of products, and customer activity. The first step involved data preprocessing, where I handled missing values, encoded categorical variables, and normalized numerical features.

Next, I performed exploratory data analysis (EDA) to understand customer behavior patterns and identify key factors influencing churn. This helped in selecting the most relevant features for model training.

For model building, I used Python with Scikit-learn and applied classification algorithms such as Logistic Regression and Random Forest. The dataset was split into training and testing sets to evaluate model performance.

The model was evaluated using metrics like accuracy, precision, recall, F1-score, and confusion matrix, and improvements were made through parameter tuning and feature optimization.

This project strengthened my understanding of customer behavior analysis, classification models, and end-to-end machine learning workflow, especially in a real-world business use case like banking.

**TECH_STACK:** `Programming Language: Python Libraries: Pandas, NumPy, Scikit-learn, Matplotlib, Seaborn Machine Learning Models: Logistic Regression, Random Forest, Decision Tree Data Processing: Data cleaning, encoding, feature scaling, feature selection Model Evaluation: Accuracy, Precision, Recall, F1-score, Confusion Matrix Frontend / Deployment: Streamlit (interactive web app) Environment: Jupyter Notebook / VS Code Version Control: Git & GitHub`
